## What this is

A macOS-only background agent app (`LSUIElement`, no Dock icon) that replaces the system window switcher with two keyboard switchers:

- **Cmd+Tab** → switch between windows of **all apps** (replaces the native app switcher).
- **Option+Tab** → switch between windows of the **current app only**.

Both show a borderless text-list overlay (app icon + window title per row), cycle with Tab / Shift+Tab while the modifier is held, and focus the highlighted window on release. Order is most-recently-used (MRU), so Tab-once returns to the previous window. Personal tool: no code signing, notarization, or settings UI. Distributed from source via a Homebrew tap (`brew tap Zappendusta/tab-switch && brew install tab-switch`); see `docs/superpowers/specs/2026-06-27-homebrew-distribution-design.md`.

## Commands

```bash
swift build                              # build
swift test                               # run all tests (TabSwitchCore only)
swift test --filter MRUListTests         # run one test class
swift test --filter MRUListTests/testOrderByMRU   # run one test method
./scripts/make-app.sh [debug|release]    # build + bundle into tab-switch.app
```

Running the app: `swift run TabSwitchApp` works but the event tap needs the **binary** to be granted Accessibility. The normal loop is `./scripts/make-app.sh && open tab-switch.app`, then grant Accessibility in System Settings. The bundle (`tab-switch.app/`) and `.build/` are gitignored and regenerated by the script.

## Architecture

Two SwiftPM targets, split by testability:

- **TabSwitchCore** — pure value/logic types, **no AppKit or Accessibility imports**, so it is unit-testable. This is the only target with tests. Keep platform types out of it.
- **TabSwitchApp** — the executable; all AppKit / Accessibility (AX) / CoreGraphics integration lives here.

### Core types (TabSwitchCore)

- `WindowInfo` / `WindowID` — plain value description of a window. `WindowID.raw` wraps the stable `CGWindowID` as a string.
- `MRUList` — most-recently-used ordering. `recordFocus` moves an id to front; `order` sorts a window list by MRU (unseen windows keep input order, sorted last); `prune` drops closed ids.
- `Scope` — `.allApps` / `.activeApp`; `filter` narrows a window list (activeApp keeps only the frontmost pid).
- `SwitcherState` — selection state for one session. Pre-selects index 1 (the previous window) when 2+ windows exist; `next`/`prev` wrap around.

### App wiring (TabSwitchApp)

`main.swift` checks Accessibility trust before starting; if missing, it prompts and polls every second until granted, then starts without relaunch.

`AppController` is the hub implementing `HotkeyTapDelegate`. One switch cycle:

1. `HotkeyTap` (a `CGEventTap` on `keyDown` + `flagsChanged`) detects Cmd/Option+Tab, **consumes** the event (returns `nil`) so the system switcher never sees it, and emits semantic delegate calls: `hotkeyOpen(scope:reverse:)`, `hotkeyNext`, `hotkeyPrev`, `hotkeyCommit`, `hotkeyCancel`.
2. On open, `AppController` enumerates windows via `AXWindowSource`, prunes/orders them through `MRUList`, builds a `SwitcherState`.
3. Further Tab → `next`; Shift+Tab → `prev`. **Modifier release** (`flagsChanged` with no Cmd/Option) → `hotkeyCommit` → `Focuser` raises + activates the window.
4. Esc → cancel.

`MRUTracker` runs independently of sessions: it observes `NSWorkspace.didActivateApplication` and records the newly-focused window into the shared `MRUList`, keeping MRU order current between switches.

### Key implementation details (easy to break)

- **`AXPrivate.swift`** declares `_AXUIElementGetWindow` via `@_silgen_name` — a private but long-stable Apple API (also used by AltTab) to get a window's `CGWindowID` from its `AXUIElement`. This is the source of stable window identity; without it there is no reliable per-window id.
- **Accessibility permission gates everything** — both the event tap and AX window control. `Permissions.isTrusted()` wraps `AXIsProcessTrusted()`. A freshly built binary is a new identity to TCC and must be re-granted.
- **Self-disabling tap recovery** — `HotkeyTap.handle` re-enables itself on `tapDisabledByTimeout` / `tapDisabledByUserInput`. Don't remove this; the OS disables taps that are slow or under heavy input.
- **Shift = step backward** is detected by a *fresh* Shift press (`shift && !shiftWasDown`) in `flagsChanged`, not by holding Shift. `shiftWasDown` is seeded at open time so an already-held Shift doesn't count as a press.
- **Flash-avoidance** — the panel renders only after a ~150ms hold (`showDelay`) or immediately on the second Tab (`renderImmediately` cancels the pending work item). A quick Cmd+Tab+release just flips to the previous window with no UI flash. Preserve this when changing show/hide timing.
- **Window filtering** — `AXWindowSource.enumerate` includes only `.regular` apps, standard titled windows (skips sheets/drawers/titleless utility windows), and windows with a resolvable `CGWindowID`. It returns the live `WindowID → AXUIElement` map (`elements`) that `Focuser` needs.
